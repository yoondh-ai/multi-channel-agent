# 콘텐츠 작성 가이드라인

## 콘텐츠 작성 원칙

### 1. 데이터 기반 작성 (Data-Grounded)
- 주어진 소스 데이터에만 기반하여 작성
- 외부 지식이나 가정 최소화
- 검증 가능한 정보만 사용

### 2. 완전한 재작성 (Full Paraphrasing)
- 단순 키워드 교체 금지
- 문장 구조를 완전히 새로 구성
- 동일한 의미를 다른 방식으로 표현

### 3. 고객 가치 중심 (Customer Value First)
- 기술 설명보다 고객 혜택 우선
- "무엇을"보다 "왜 중요한지" 강조
- 구체적인 비즈니스 임팩트 제시

## 콘텐츠 작성 프로세스

### Phase 1: Thinking Process (사고 과정)

글을 쓰기 전 반드시 다음을 정리:

1. **타겟 분석**
   - 이 글의 주요 독자는 누구인가?
   - 그들의 주요 관심사는?
   - 그들의 전문성 수준은?

2. **핵심 가치 정의**
   - 이 글에서 전달할 핵심 메시지는?
   - 독자가 얻을 수 있는 가치는?
   - 왜 지금 이 내용이 중요한가?

3. **채널 특성 고려**
   - 이 채널의 특성은?
   - 적절한 길이와 형식은?
   - 톤앤매너는?

4. **차별화 포인트**
   - 다른 콘텐츠와의 차이점은?
   - 독자의 주목을 끌 요소는?
   - 행동을 유도할 방법은?

### Phase 2: Content Structure (구조 설계)

#### 블로그 포스트 구조
```
1. 훅 (Hook): 독자의 관심을 끄는 질문이나 통계
2. 문제 정의: 독자가 직면한 과제
3. 해결책 제시: MarkAny의 접근 방법
4. 구체적 예시: 실제 사례나 데이터
5. 실행 방법: 독자가 취할 수 있는 액션
6. CTA: 명확한 다음 단계
```

#### LinkedIn 포스트 구조
```
1. 인사이트: 흥미로운 발견이나 트렌드
2. 개인적 경험: 관련된 경험이나 관찰
3. 비즈니스 임팩트: 왜 중요한가
4. 질문: 독자 참여 유도
5. 해시태그: 3-5개
```

#### Twitter 스레드 구조
```
1. 훅 트윗: 강력한 주장이나 질문
2. 문제 제기: 현재 상황의 문제점
3. 해결책: 핵심 아이디어
4. 증거: 데이터나 사례
5. CTA: 행동 유도
```

#### 이메일 구조
```
1. 제목: 호기심 유발 (50자 이내)
2. 프리헤더: 제목 보완
3. 인사: 개인화된 시작
4. 가치 제안: 핵심 혜택
5. 증거: 사례나 데이터
6. CTA: 명확한 버튼
7. 서명: 연락처
```

### Phase 3: Writing (작성)

#### 문장 작성 원칙

**간결성 (Conciseness)**
```
❌ 나쁜 예:
"당사의 솔루션을 도입하신 고객사들께서는 보안 사고 발생률이 현저히 감소하는 
효과를 경험하고 계시며, 이는 통계적으로도 유의미한 결과로 나타나고 있습니다."

✅ 좋은 예:
"MarkAny 도입 후 보안 사고가 평균 73% 감소했습니다."
```

**구체성 (Specificity)**
```
❌ 나쁜 예:
"많은 기업들이 우리 솔루션을 사용합니다."

✅ 좋은 예:
"금융권 상위 10개 은행 중 8곳이 MarkAny를 사용합니다."
```

**능동태 (Active Voice)**
```
❌ 나쁜 예:
"보안 위협이 실시간으로 탐지됩니다."

✅ 좋은 예:
"AI가 보안 위협을 실시간으로 탐지합니다."
```

#### 단락 구성

**한 단락 = 한 아이디어**
- 첫 문장: 핵심 주장
- 중간 문장: 설명이나 예시
- 마지막 문장: 요약이나 전환

**예시:**
```
제로트러스트는 더 이상 선택이 아닌 필수입니다. 
최근 조사에 따르면 원격 근무 확대로 인해 보안 위협이 300% 증가했습니다. 
MarkAny의 제로트러스트 솔루션은 이러한 위협을 사전에 차단합니다.
```

### Phase 4: Refinement (다듬기)

#### 체크리스트

**명확성 (Clarity)**
- [ ] 전문 용어에 설명이 있는가?
- [ ] 문장이 이해하기 쉬운가?
- [ ] 핵심 메시지가 명확한가?

**정확성 (Accuracy)**
- [ ] 모든 데이터가 정확한가?
- [ ] 출처가 명확한가?
- [ ] 과장된 표현이 없는가?

**일관성 (Consistency)**
- [ ] 브랜드 톤에 맞는가?
- [ ] 용어 사용이 일관적인가?
- [ ] 메시지가 일관적인가?

**효과성 (Effectiveness)**
- [ ] 타겟 독자에게 적합한가?
- [ ] 행동을 유도하는가?
- [ ] 차별화 포인트가 있는가?

## 채널별 세부 가이드

### 블로그 포스트

**길이**: 1,500-2,500자
**톤**: 전문적, 교육적
**구조**: 논리적, 체계적

**필수 요소:**
- SEO 키워드 (자연스럽게 3-5회)
- 소제목 (H2, H3)
- 불릿 포인트
- 구체적 예시
- 이미지 삽입 위치 제안
- 관련 글 링크
- CTA

**예시 템플릿:**
```
# [숫자/질문] + [핵심 키워드]
예: "제로트러스트 도입 시 반드시 고려해야 할 5가지"

## 도입부 (200자)
- 현재 상황/문제 제기
- 이 글에서 다룰 내용 예고

## 본문 (1,000-1,500자)
### 소제목 1
- 핵심 포인트
- 구체적 예시
- 데이터/통계

### 소제목 2
[반복]

## 실행 방법 (300자)
- 구체적 액션 아이템
- 단계별 가이드

## 결론 (200자)
- 핵심 메시지 요약
- CTA
```

### LinkedIn 포스트

**길이**: 800-1,200자
**톤**: 전문적, 인사이트 중심
**구조**: 인사이트 → 사례 → 질문

**필수 요소:**
- 첫 2줄에 핵심 메시지
- 개인적 경험이나 관찰
- 구체적 데이터
- 참여 유도 질문
- 해시태그 3-5개

**예시 템플릿:**
```
[강력한 주장이나 질문]
[핵심 인사이트]

최근 [경험/관찰]을 통해 [발견]했습니다.

[구체적 사례나 데이터]

이것이 중요한 이유:
• [이유 1]
• [이유 2]
• [이유 3]

여러분의 경험은 어떠신가요?

#키워드1 #키워드2 #키워드3
```

### Twitter 스레드

**길이**: 3-7개 트윗
**톤**: 간결, 임팩트
**구조**: 훅 → 핵심 → CTA

**필수 요소:**
- 첫 트윗에 강력한 훅
- 각 트윗 280자 이내
- 이모지 활용
- 해시태그 2-3개
- 마지막 트윗에 CTA

**예시 템플릿:**
```
[트윗 1] 🔥 [충격적 통계/질문]
#키워드1 #키워드2

[트윗 2] 문제는 이겁니다:
[문제 설명]

[트윗 3] 해결책은?
[핵심 아이디어]

[트윗 4] 실제 사례:
[구체적 예시]

[트윗 5] 지금 바로 [CTA]
[링크]
```

### 마케팅 이메일

**길이**: 500-800자
**톤**: 친근하면서 전문적
**구조**: 가치 제안 → 혜택 → CTA

**필수 요소:**
- 제목 (50자 이내, 호기심 유발)
- 프리헤더 (제목 보완)
- 개인화 인사
- 명확한 가치 제안
- 구체적 혜택
- 단일 CTA
- 모바일 최적화

**예시 템플릿:**
```
제목: [숫자/질문] + [혜택]
프리헤더: [제목 보완, 추가 정보]

안녕하세요, [이름]님

[개인화된 인사/맥락]

[핵심 가치 제안]

이렇게 도움이 됩니다:
✓ [혜택 1]
✓ [혜택 2]
✓ [혜택 3]

[사회적 증거: 고객 수, 만족도 등]

[CTA 버튼]

감사합니다.
[서명]
```

## 금지 사항

### 절대 하지 말 것

1. **표절**: 원문을 그대로 복사
2. **과장**: 검증 불가능한 주장
3. **일반화**: "모든", "항상", "절대"
4. **비방**: 경쟁사 직접 언급
5. **강압**: "반드시", "무조건"

### 신중히 사용

1. **전문 용어**: 설명 없이 사용 금지
2. **유행어**: 맥락 없이 사용 금지
3. **통계**: 출처 없이 사용 금지
4. **사례**: 허가 없이 고객사 언급 금지

## 품질 기준

### 최소 기준 (Must Have)
- 문법 오류 없음
- 브랜드 톤 준수
- 핵심 메시지 명확
- CTA 포함

### 우수 기준 (Should Have)
- 구체적 데이터 포함
- 실제 사례 포함
- 독창적 인사이트
- 독자 참여 유도

### 탁월 기준 (Nice to Have)
- 차별화된 관점
- 예상치 못한 인사이트
- 강력한 스토리텔링
- 바이럴 가능성
