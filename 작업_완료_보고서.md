#### 2.2 최소 글자 수 강제 적용

**기준**:

- Twitter: 각 트윗 200자 이상

**효과**: 짧고 성의 없는 콘텐츠 생성 방지

#### 2.3 마케팅 프레임워크 완벽 적용

**구현 위치**: `src/gemini_generator.py` - `_get_framework_instructions()` 메서드

**프레임워크별 상세 지침**:

**PAS (Problem-Agitate-Solution)**
- Problem: 150자 이상 - 고객이 직면한 구체적 문제
- Agitate: 150자 이상 - 문제 방치 시 위험성 강조
- Solution: 200자 이상 - 제품/서비스의 해결 방법

**AIDA (Attention-Interest-Desire-Action)**
- Attention: 100자 이상 - 충격적 통계나 질문
- Interest: 150자 이상 - 흥미로운 사실이나 트렌드
- Desire: 150자 이상 - 제품 사용 후 긍정적 변화
- Action: 100자 이상 - 명확한 행동 유도

**BAB (Before-After-Bridge)**
- Before: 150자 이상 - 현재 상황의 문제점
- After: 150자 이상 - 개선된 이상적 상황
- Bridge: 200자 이상 - 그 미래에 도달하는 방법

**FAB (Features-Advantages-Benefits)**
- Features: 150자 이상 - 제품의 구체적 기능
- Advantages: 150자 이상 - 경쟁사 대비 우위
- Benefits: 200자 이상 - 고객이 얻는 실질적 가치

**효과**: 체계적이고 설득력 있는 마케팅 콘텐츠 생성

#### 2.4 풍부한 콘텐츠 작성 강제

**구현 내용**:

**구체적 사례 요구**:
- "예를 들어, A사는..." 형식의 실제 사례
- 구체적인 상황 묘사
- 독자가 공감할 수 있는 스토리

**수치와 데이터 요구**:
- "평균 73% 감소" 같은 구체적 수치
- "3개월 만에 ROI 250%" 같은 성과
- "업계 68%가 직면한 문제" 같은 통계

**감정적 연결 요구**:
- 독자의 고민과 걱정 공감
- 해결 후의 긍정적 미래 제시
- "이제 안심하고..." 같은 감정 표현

**명확한 구조 요구**:
- 도입 → 본론 → 결론
- 각 섹션마다 소제목
- 논리적 흐름

**효과**: 키워드만 반복하는 저품질 콘텐츠 방지

### 4. Gemini 모델 자동 선택 🤖

**문제**: Gemini API 404 NOT_FOUND 오류 발생

**해결**: `src/gemini_generator.py` - `_select_best_model()` 메서드 구현

**동작**:
1. `genai.list_models()`로 사용 가능한 모델 목록 조회
2. 우선순위에 따라 최적 모델 선택:
   - gemini-2.0-flash-exp (최신 실험 버전)
   - gemini-2.0-flash (2.0 안정 버전)
   - gemini-1.5-flash-latest (1.5 최신 버전)
   - gemini-1.5-flash (1.5 기본 버전)
   - gemini-1.5-flash-002 (1.5 특정 버전)
   - gemini-pro (폴백 옵션)
3. 사용 가능한 모델이 없으면 기본 모델 사용

**효과**: 모델 버전 변경이나 지역별 가용성 차이에도 안정적으로 작동
