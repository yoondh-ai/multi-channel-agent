**입력 데이터:**
```

핵심 특징:
• 완벽한 보안 아키텍처로 내부/외부 위협 차단
• 클라우드 네이티브 설계로 빠른 도입
• AI 기반 실시간 위협 탐지 및 대응
• 간편한 관리 콘솔로 운영 부담 최소화
• 24/7 전문가 지원 서비스

타겟 키워드: 제로트러스트, 클라우드 보안, 네트워크 보안

추가 정보:

전통적인 경계 기반 보안으로는 더 이상 충분하지 않습니다.

타겟 독자: C-레벨 임원
말투: 전문적
프레임워크: PAS
```

**예상 결과:**
- ✅ 블로그: 1,500자 이상
- ✅ LinkedIn: 800자 이상  
- ✅ Twitter: 3-7개 트윗, 각 200자 이상
- ✅ 이메일: 500자 이상

**품질 체크:**
- [ ] Problem (문제 제기) 섹션이 150자 이상인가?
- [ ] Agitate (고통 증폭) 섹션이 150자 이상인가?
- [ ] Solution (해결책) 섹션이 200자 이상인가?
- [ ] 구체적인 수치나 통계가 포함되어 있는가?
- [ ] 스토리텔링 요소가 있는가?
- [ ] 키워드만 반복하지 않고 풍부한 서술인가?

### 1. 글자 수 확인

```python
# 생성된 콘텐츠를 복사하여 글자 수 확인
content = """[생성된 콘텐츠 붙여넣기]"""
```

**기준:**

- 이메일: 500자 이상 ✅

### 2. 프레임워크 구조 확인

**PAS 체크:**
```
[ ] Problem (문제 제기) - 150자 이상
    - 고객이 직면한 구체적 문제 상황
    - 공감할 수 있는 사례
    
[ ] Agitate (고통 증폭) - 150자 이상
    - 문제를 방치했을 때의 위험
    - 구체적인 손실 (시간, 비용, 기회)
    
[ ] Solution (해결책) - 200자 이상
    - 제품/서비스가 어떻게 해결하는지
    - 구체적인 기능과 효과
    - 실제 성과 수치
```

**AIDA 체크:**
```
[ ] Attention (주목) - 100자 이상
    - 충격적 통계나 질문
    - 호기심 유발
    
[ ] Interest (관심) - 150자 이상
    - 흥미로운 사실이나 트렌드
    - 독자와의 관련성 강조
    
[ ] Desire (욕구) - 150자 이상
    - 제품 사용 후의 긍정적 변화
    - 구체적 혜택 나열
    
[ ] Action (행동) - 100자 이상
    - 명확한 다음 단계
    - 긴급성 조성
```

## 🐛 문제 해결

### 문제 1: 콘텐츠가 너무 짧음 (500자 미만)

**원인:**
- Gemini가 지시를 무시하고 짧게 생성

**해결:**
1. 더 구체적인 입력 제공
2. "추가 정보" 필드에 더 많은 컨텍스트 입력
3. 프레임워크를 명확히 선택

**확인:**
- System Instruction에 최소 글자 수가 명시되어 있는지 확인
- User Prompt에 프레임워크별 상세 지침이 포함되어 있는지 확인

### 문제 2: 키워드만 반복됨

**원인:**
- AI가 창의적으로 작성하지 않고 키워드만 나열

**해결:**
1. "추가 정보"에 스토리나 사례 추가
2. 다른 프레임워크 시도 (PAS → AIDA)
3. 말투를 "감성적" 또는 "스토리텔링"으로 변경

**확인:**
- System Instruction에 "단순 키워드 나열 금지" 명시 확인
- "스토리텔링 기법 활용" 지시 확인

### 문제 3: Gemini API 404 오류

**원인:**
- 모델명이 잘못되었거나 사용 불가능

**해결:**
- 자동 모델 선택 기능이 작동하는지 확인
- `_select_best_model()` 메서드가 올바르게 구현되어 있는지 확인

**확인:**
```python
# 사용 가능한 모델 확인
import google.generativeai as genai
genai.configure(api_key="YOUR_API_KEY")
models = genai.list_models()
for model in models:
    if 'generateContent' in model.supported_generation_methods:
        print(model.name)
```

### 문제 4: UI 레이아웃이 무너짐

**원인:**
- 컬럼 비율이 잘못 설정됨

**해결:**
- `app.py` 라인 186 확인: `st.columns([1, 1.2], gap="large")`
- 모든 입력 폼이 `left_col` 안에 있는지 확인
- 모든 결과 표시가 `right_col` 안에 있는지 확인
