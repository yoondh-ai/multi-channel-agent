# 채널별 콘텐츠 가이드라인

## 📝 네이버 블로그

### 최적 길이
- 1,500 - 2,500자
- 읽기 시간: 5-7분

### 구조
1. **도입부** (200-300자)
   - 문제 제기 또는 질문
   - 독자의 관심 유발
   - 본문 내용 예고

2. **본문** (1,000-1,800자)
   - 3-5개 섹션으로 구분
   - 각 섹션에 소제목 (H2, H3)
   - 불릿 포인트 활용
   - 이미지/표 삽입 위치 제안

3. **결론** (200-300자)
   - 핵심 내용 요약
   - 실행 가능한 조언
   - CTA

### SEO 최적화
- **제목 태그**: 30-60자, 핵심 키워드 포함
- **메타 설명**: 120-160자, 클릭 유도
- **키워드 밀도**: 1-2%, 자연스럽게 배치
- **내부 링크**: 관련 포스트 2-3개 연결
- **이미지 alt 텍스트**: 키워드 포함 설명

### 스타일
- 문단: 3-4줄 이내
- 문장: 간결하고 명확하게
- 전문 용어: 쉬운 설명 추가
- 이모지: 제목과 소제목에 적절히 사용

### 필수 요소
```
✅ 매력적인 제목 (숫자, 질문, 혜택 포함)
✅ 썸네일 이미지 제안
✅ 목차 (긴 글의 경우)
✅ 핵심 내용 하이라이트
✅ 관련 링크 (공식 사이트, 자료)
✅ 태그 10-15개
```

### 예시 구조
```markdown
# [숫자] + [혜택] + [키워드] (예: "5분 만에 이해하는 제로트러스트 보안")

## 도입
[문제 상황 제시]

## 목차
1. 제로트러스트란?
2. 왜 필요한가?
3. 도입 방법
4. 성공 사례
5. 시작하기

## 본문
### 1. 제로트러스트란?
[설명 + 이미지]

### 2. 왜 필요한가?
• 포인트 1
• 포인트 2
• 포인트 3

[계속...]

## 결론
[요약 + CTA]

---
태그: #제로트러스트 #보안 #클라우드보안 ...
```

## 💼 LinkedIn

### 최적 길이
- 800 - 1,200자
- 읽기 시간: 3-4분

### 구조
1. **훅** (첫 2-3줄)
   - 강력한 문장으로 시작
   - 스크롤 멈추게 만들기
   - 질문 또는 놀라운 사실

2. **본문**
   - 개인적 경험 또는 인사이트
   - 구체적인 예시와 데이터
   - 스토리텔링

3. **마무리**
   - 핵심 메시지 강조
   - 질문으로 참여 유도
   - 해시태그 3-5개

### 톤 & 스타일
- **전문적이면서 개인적**
- 1인칭 시점 활용
- 경험과 인사이트 공유
- 데이터와 사실 기반

### 포맷팅
```
✅ 짧은 문단 (2-3줄)
✅ 이모지 적절히 사용 (과하지 않게)
✅ 공백으로 가독성 확보
✅ 중요 문장 강조 (볼드)
```

### 참여 유도
- 질문으로 마무리
- 의견 요청
- 경험 공유 유도
- 태그 활용 (@회사명, @인물)

### 해시태그 전략
- 3-5개 (너무 많으면 스팸)
- 업계 관련 태그
- 트렌딩 태그
- 브랜드 태그

### 예시
```
최근 한 금융기관의 보안 담당자와 대화를 나눴습니다.

"제로트러스트를 도입하고 싶은데, 어디서부터 시작해야 할지 모르겠어요."

이 질문을 받고 깨달았습니다. 
많은 기업이 같은 고민을 하고 있다는 것을.

[본문 전개...]

여러분의 조직은 어떤 단계에 있나요?
댓글로 경험을 공유해주세요.

#제로트러스트 #사이버보안 #디지털전환 #MarkAny
```

- 3-7개 트윗
- 각 트윗 250-280자 (여유 공간 확보)

- 간결하고 명확하게

## 📧 이메일

### 제목 (Subject Line)
- 30-50자
- 개인화 ([이름]님, [회사]님)
- 혜택 명확히
- 긴급성 또는 호기심
- A/B 테스트 권장

### 프리헤더
- 40-100자
- 제목 보완
- 추가 정보 제공

### 본문 구조
1. **인사** (1-2줄)
   - 개인화된 인사
   - 관계 언급

2. **문제/기회 제시** (2-3줄)
   - 독자의 pain point
   - 시의성 있는 이슈

3. **솔루션 제시** (3-5줄)
   - 제품/서비스 소개
   - 핵심 혜택 3가지
   - 차별점

4. **증거** (2-3줄)
   - 데이터, 사례
   - 고객 후기
   - 인증/수상

5. **CTA** (명확하게)
   - 버튼 형태
   - 행동 지향적
   - 하나의 주요 CTA

6. **마무리** (1-2줄)
   - 감사 인사
   - 추가 정보 제공

### 스타일
```
✅ 짧은 문단 (2-3줄)
✅ 불릿 포인트 활용
✅ 모바일 최적화
✅ 개인화 요소
✅ 시각적 계층 구조
```

### CTA 버튼
- 명확한 행동 지시
- 대비되는 색상
- 충분한 크기
- 클릭 가능 영역 확보

### 예시
```
제목: [이름]님, 보안 위협으로부터 회사를 지키는 3가지 방법

프리헤더: 5분 투자로 1년 동안 안심하세요

---

안녕하세요, [이름]님
```
